# Typography

Hierarchy is carried by typography, not by paint. Iowan Old Style for prose, Berkeley Mono for code. No color is used to distinguish levels — only size, weight, italic, and tracking. The drop cap on this very paragraph drops four lines and hangs slightly into the gutter; it appears on the first paragraph after every H1.

## Heading two — a section opener

A section starts with an H2. The margin above is the breath before a turn. The weight is heavy enough to read as a beat, the tracking tightened so it doesn't shout.

### Heading three — a subsection

H3 is the rhythm-mate to body text. Same family, only slightly larger, slightly heavier. It sits inside an H2's argument without competing with it.

#### Heading four is italic

Used sparingly. For an aside, a subhead within an essay, a slot inside an H3 that needs its own label.

##### Heading five
Quieter still — for technical labels and parameter names.

###### HEADING SIX IS MONO

Reserved for inline metadata — dates, IDs, system breadcrumbs.

## Inline texture

**Bold** is genuine ink. *Italic* is the speaking voice. ~~Strikethrough~~ marks a kill. `inline code` is amber on a hairline tint, the texture of mechanism. Combinations like ***bold italic*** and **bold with `code` inside** read cleanly.

Links underline softly and brighten on hover — for example, [the MarkView repository](https://github.com/abgnydn/markview) or a [section anchor](#heading-two--a-section-opener) inside this very file.

## Quotation
